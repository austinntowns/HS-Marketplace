From src/components/kpi/KpiSection.tsx:
```typescript
interface Location {
  id: string
  name: string
  type: 'suite' | 'flagship' | 'territory'
}

interface KpiSectionProps {
  locationId?: string          // Single location ID for suite/flagship
  bundleLocations?: Location[] // Multiple locations for bundle listings
  listingType: 'suite' | 'flagship' | 'territory' | 'bundle'
}
```

Key mapping note: KpiSection.Location.type ('suite' | 'flagship' | 'territory') maps from
ListingDetailLocation.locationType ('salon' | 'territory'). Salon locations should be
typed based on the parent listing.type (suite or flagship). For bundle listings, each
location's type comes from its locationType: 'salon' locations in a bundle are mapped
using the listing type if it's not a bundle — but for a bundle, salon locations should
pass as 'suite' or 'flagship' based on what's available. Since ListingDetailLocation
only has 'salon' or 'territory', map 'salon' → use listing type for singles, or default
to 'suite' for bundle sub-locations (conservative choice; KpiSection only uses type to
filter out territories).
