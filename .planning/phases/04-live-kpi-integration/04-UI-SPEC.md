## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon-to-text gap inside KPI badge, MoM arrow gap |
| sm | 8px | KPI card internal padding (label-to-value), table cell vertical padding |
| md | 16px | KPI card padding, modal body padding, chart margin |
| lg | 24px | Gap between KPI cards in the 4-column row, section padding |
| xl | 32px | KPI section top margin (below seller financials), modal padding |
| 2xl | 48px | Major section break between seller financials and KPI section |
| 3xl | 64px | Page-level vertical rhythm (not used inside KPI components) |

Exceptions:
- KPI card minimum height: 120px (ensures the primary number, MoM line, and timestamp all fit without crowding)
- Chart height inside modal: 240px fixed (per Recharts `ResponsiveContainer` height — matches RESEARCH.md Pattern 3)
- Touch target for card click: 44px minimum height override on mobile (accessibility)

## Typography

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Body | 14px | 400 (regular) | 1.5 |
| Label | 12px | 500 (medium) | 1.4 |
| Heading | 20px | 600 (semibold) | 1.2 |
| Display | 28px | 700 (bold) | 1.1 |

**Applied to KPI components (source: Claude's Discretion, consistent with existing codebase):**
- KPI section heading ("Live Performance Data"): 20px, weight 600, line-height 1.2 — matches existing `text-lg font-semibold` pattern
- KPI card metric name (e.g. "Revenue"): 12px, weight 500, color `text-gray-500` — matches existing label style
- KPI card primary value (e.g. "$45,230"): 28px, weight 700, color `text-gray-900` — display role
- KPI card MoM change (e.g. "+12% ↑"): 14px, weight 400, color `text-gray-600` — body role
- KPI card freshness timestamp (e.g. "Updated 2 hours ago"): 12px, weight 400, color `text-gray-400` — label role
- Bundle table header: 12px, weight 500, uppercase, `text-gray-500` — matches existing `th` style
- Bundle table cell: 14px, weight 400, `text-gray-900` — body role
- Modal title (e.g. "Revenue — 12-Month Trend"): 20px, weight 600 — heading role
- Tooltip text inside chart: 14px, weight 400 — body role (Recharts default, styled via `contentStyle`)

## Component Inventory

These are the components Phase 4 must implement. Referenced from RESEARCH.md §Architecture Patterns.

### KpiSection (Server Component)
- Conditionally renders: hidden entirely when `fetchLocationKpi` returns null (API down) or listing type is territory
- Renders section heading: `<h2>Live Performance Data</h2>` or `<h2>Live Performance Data ({N} locations)</h2>` for bundles
- Wraps `KpiContent` in `<Suspense fallback={null}>` — listing shell renders immediately, KPI streams in
- Layout: `space-y-6` between heading and card row

### KpiCard (Client Component — needs onClick)
- Dimensions: `min-h-[120px]` width auto (flex child in 4-column grid), `rounded-lg`, `border border-gray-200`, `bg-white`, `shadow-sm`
- Cursor: `cursor-pointer` with `hover:border-pink-300 hover:shadow-md` transition
- Badge (top-right of card): pill shape, `bg-pink-600 text-white text-xs px-2 py-0.5 rounded-full` containing SVG checkmark + "Live" text
- Metric name: `text-xs font-medium text-gray-500 uppercase tracking-wide`
- Primary value: `text-3xl font-bold text-gray-900` (28px maps to Tailwind `text-3xl`)
- MoM change: `text-sm text-gray-600` with Unicode arrow — "↑ +12%" or "↓ -3%" — arrow and percentage on same line, no color change per locked decision
- Freshness timestamp: `text-xs text-gray-400` — "Updated 2 hours ago"
- On click: opens `KpiTrendModal` with this KPI's 12-month trend data
- Partial data: card is omitted entirely when its KPI field is absent — grid collapses from 4 to available count using `grid-cols-{n}` or auto-fit

### KpiTrendModal (Client Component)
- Trigger: `KpiCard` onClick
- Pattern: fixed overlay modal (not drawer) — `fixed inset-0 z-50 flex items-center justify-center`
- Scrim: `bg-black/50` (semi-transparent) — click scrim to close
- Modal panel: `bg-white rounded-xl shadow-xl w-full max-w-2xl mx-4 p-8`
- Animation (Claude's Discretion): fade-in + scale-up — `transition-all duration-150 ease-out` from `opacity-0 scale-95` to `opacity-100 scale-100`
- Close button: `X` in top-right corner, `text-gray-400 hover:text-gray-600`
- Title: `text-xl font-semibold text-gray-900` — "{KPI Name} — 12-Month Trend"
- Contains: `KpiTrendChart` at 240px height

### KpiTrendChart (Client Component — `'use client'` required for Recharts)
- Library: Recharts 3.8.0 — `LineChart`, `Line`, `XAxis`, `YAxis`, `CartesianGrid`, `Tooltip`, `ResponsiveContainer`
- Height: 240px via `<ResponsiveContainer width="100%" height={240}>`
- Line: `type="monotone"`, `dot={false}`, `strokeWidth={2}`, stroke `#db2777` (pink-600)
- Grid: `strokeDasharray="3 3"` in `#e5e7eb` (gray-200)
- X axis: month labels (e.g. "Oct", "Nov") — 12 ticks, `tick={{ fontSize: 12, fill: '#6b7280' }}`
- Y axis: formatted values (dollar amounts use `$` prefix, percentages use `%` suffix, counts use locale string) — `tick={{ fontSize: 12, fill: '#6b7280' }}`
- Tooltip: `contentStyle={{ fontSize: '14px', borderRadius: '6px', border: '1px solid #e5e7eb' }}` — format: "{value} • {month}" per CONTEXT.md locked decision (e.g. "$45,230 • Oct 2025")
- No legend, no reference line

### BundleKpiTable (Client Component — needs sort state)
- Layout: `overflow-x-auto` wrapper, `min-w-full` table
- Header row: `bg-gray-50`, cells `px-6 py-3 text-left text-xs font-medium text-gray-500 uppercase` — matches existing admin table style
- Sortable columns: all four KPI columns + location name — click header to sort asc/desc; active column shows `↑` or `↓` in `text-pink-600` beside header label
- Default sort: location name ascending
- Body rows: `bg-white divide-y divide-gray-200`, `px-6 py-4 text-sm text-gray-900`
- 5+ locations: `max-h-[400px] overflow-y-auto` on tbody wrapper — vertical scroll, no pagination
- Row click: inline expand — the clicked row expands below to reveal `KpiTrendChart` at 200px height; only one row expanded at a time; clicking again collapses; expanded row has `bg-gray-50` background
- Territories in bundle: separate `<section>` below the table with heading "Unopened Territories" and a plain list of territory names — not in the sortable table

### BundleOverlayChart (Client Component)
- Same Recharts structure as `KpiTrendChart` but renders one `<Line>` per location
- Each line gets a color from PALETTE array (cycling if >8 locations)
- Legend: `<Legend>` component below chart — location name + color swatch — `wrapperStyle={{ fontSize: '12px' }}`
- Height: 280px (taller than single-location chart to accommodate legend)
- Accessible via: KPI section has a "View all locations" button above the bundle table; clicking opens `KpiTrendModal` with `BundleOverlayChart` instead of `KpiTrendChart`

## Interaction Contract

| Interaction | Behavior |
|-------------|----------|
| KPI card click | Opens `KpiTrendModal` with 12-month line chart for that KPI |
| Modal scrim click | Closes modal |
| Modal X button click | Closes modal |
| Escape key | Closes open modal |
| Bundle table column header click | Sorts table by that column; toggles asc/desc; active column highlighted in pink-600 |
| Bundle table row click | Inline expands/collapses that row's trend chart |
| "View all locations" click | Opens overlay chart modal with all location lines |

## Data States

| State | KPI Section Behavior |
|-------|---------------------|
| API fully available | Full 4-card row renders; each card shows last-month value, MoM change, freshness timestamp, badge |
| API completely down | `fetchLocationKpi` returns null; `KpiSection` returns null; section is hidden; only seller financials shown |
| Partial data (1–3 KPIs available) | Available cards render; missing cards are omitted; grid collapses (e.g. 3 cards use `grid-cols-3`) |
| Stale data (cached) | Cached data renders normally; freshness timestamp shows actual age (e.g. "Updated 47 minutes ago") |
| Territory listing | KPI section hidden entirely — territories have no operational data |
| Suspense loading | `fallback={null}` — no skeleton, no spinner; page renders without KPI section until data streams in |

**Source:** CONTEXT.md locked decisions §Unavailable State and §KPI Card Layout

**KPI metric display names (canonical labels for cards and modal titles):**

| Internal field | Display label |
|----------------|---------------|
| revenue | Revenue |
| newClients | New Clients |
| bookings | Bookings |
| membershipConversion | Membership Conversion |

**MoM change formatting:**
- Positive: "↑ +{N}%" (e.g. "↑ +12%")
- Negative: "↓ -{N}%" (e.g. "↓ -3%")
- Zero: "→ 0%" (no change)
- Font color for all three: `text-gray-600` — per CONTEXT.md locked decision (neutral, no green/red; arrow direction conveys meaning)

**Primary value formatting by KPI:**
- Revenue: USD currency, no cents — "$45,230"
- New Clients: integer with locale commas — "1,204"
- Bookings: integer with locale commas — "3,890"
- Membership Conversion: percentage, one decimal — "45.2%"
